## Maintainer Checklist

- Review issues and pull requests with respect and clear rationale.
- Keep `README.md`, `docs/`, examples, and tests in sync.
- Run release checks before publishing.
- Rotate any credential that appears in public artifacts.

## Adding Maintainers

New maintainers should demonstrate consistent contributions, good judgment around security boundaries, and respect for the project's focused scope.
